# NTE（异环）字体投放区

把你手上那个**可能是 NTE（异环 / Neverless to Everless）游戏内自创文字**的字体文件放在这个目录下。

## 投放要求

1. 文件名建议保留原始命名（便于追溯出处），常见格式：`.ttf` / `.otf` / `.ttc` / `.woff` / `.woff2` 都可以。
2. 如果有多个候选字体文件，**全部放进来**，我会逐个分析，挑出真正的"自创文字"那一个（很多游戏的字体包里会混着 UI 字体、日韩 fallback、emoji 等无关字体）。
3. 如果你还能顺手提供 1-3 张**真实 NTE 游戏截图（带自创文字）**，请放到 `../../samples/nte/` 下，命名随意（如 `screenshot_01.png`），用于和 TTF 渲染结果做形状比对。

## 我会做什么分析（Phase 0 的核心任务）

放进来之后告诉我，我会执行以下步骤：

1. **基础元信息提取**：字体名、Family、Foundry、字符总数、Unicode 覆盖范围
2. **字符表导出**：把字体内所有 glyph 渲染成 PNG 拼成一张总览图
3. **可疑度评分**：判断这个字体是不是"自创文字"
   - 如果是常规拉丁/CJK 字体 → 大概率不是
   - 如果 glyph 占用的 Unicode 区段是 PUA（Private Use Area, U+E000-U+F8FF）→ 高度可疑是自创文字
   - 如果 glyph 形状和真实游戏截图能对上 → 确认是
4. **映射推测**：如果确认是自创文字，尝试推测 glyph → 英文字母的对应关系
   - 看字体内是否有 `cmap` 表把这些 glyph 映射到了 A-Z 的 Unicode 码位（最理想情况）
   - 如果没有，则需要靠真实截图标注辅助

## 产出

分析完后，我会在 `../../docs/nte-font-report.md` 写一份报告，包含：

- 字体是否可用的结论
- 字符总览图
- 映射方案（自动/半自动/手动）
- 进入 Phase 1 训练流水线前的准备清单
